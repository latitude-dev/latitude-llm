1. Verify required secrets in values.yaml under secretEnv:
   - DATABASE_URL, NEXT_SERVER_ACTIONS_ENCRYPTION_KEY, GOOGLE_CLIENT_SECRET,
     DEFAULT_PROVIDER_API_KEY, SMTP_*, SENTRY_*, AWS_*, etc.

2. Set ingress hosts in values.yaml (ingress.hosts):
   - web: app-latitude.example.com
   - gateway: gateway-latitude.example.com
   - websockets: ws-latitude.example.com

3. Storage: the chart mounts a shared PVC to:
   - /app/storage/files
   - /app/apps/web/public/files
   Ensure your storage class supports ReadWriteMany or disable sharedStorage.

4. Install/upgrade:
   helm upgrade --install latitude ./charts/latitude \
     --namespace latitude --create-namespace \
     -f your-values.yaml

5. Migrations:
   A pre-install/upgrade Job runs database migrations using ghcr.io/latitude-dev/migrations.

6. Images:
   - ghcr.io/latitude-dev/web:latest
   - ghcr.io/latitude-dev/gateway:latest
   - ghcr.io/latitude-dev/workers:latest
   - ghcr.io/latitude-dev/websockets:latest
   - ghcr.io/latitude-dev/migrations:latest
